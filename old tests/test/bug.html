Failing Cases - Missing Spaces in Numeric Parameters

`from45deg` becomes `conic-gradient(from45deg, ...)` which is wrong
`at50%50%` becomes `conic-gradient(at50%50%, ...)` which is wrong
`from90deg,at30%80%` Also a parameter that doesn't work.
`from45deg,atTopLeft` Also a parameter that doesn't work.

Issue: CSSS camelCase conversion fails with numeric values (no uppercase letters to convert)

Valid Conic Syntax: conic-gradient([from <angle>] [at <position>], colors...)

Root Cause: Regex /[A-Z]/g only converts uppercase letters, missing numeric combinations

Fix Needed: Enhanced parameter conversion to handle digits + units (deg, %, px, etc.)
