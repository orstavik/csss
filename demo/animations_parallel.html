Parallel Multi-Property Animations

These demos test animations with multiple properties that run in parallel. They have the same or different durations but no complex percent-based keyframes.

Scale + Opacity

Two properties, same duration (0.5s), synchronized timing.

Try Me
$scale(1)$opacity(1)$to(scale(1.5),opacity(0.5))$animation(0.5s)

Rotate + Translate

Transform composition: rotation and translation together.

Spin
$rotate(0deg)$translateX(0)$to(rotate(180deg),translateX(50px))

Background + Border

Different durations: background 0.3s, border 0.6s. Parallel but not synced.

Colors
$bg(#f093fb)$Border(0)$to(bg(#4facfe),border(5px,#00f2fe))

Triple Property

Opacity, scale, and rotation - all with same duration.

Triple
$opacity(1)$scale(1)$rotate(0)$to(0.4,0.8,-90deg)

Complex Parallel

Multiple properties, different durations and easings. Tests if synchronization matters.

Complex
$transform(scale(1),rotate(0))$opacity(1)$bg(#f093fb) $to(scale(1.3),rotate(45deg),opacity(0.7),bg(#fa709a))

Research Notes: Parallel Animations

Question: How often are different properties truly synced beyond just duration?

Observation: Many multi-property animations use parallel timing (just from/to) without complex keyframe synchronization. Different properties often benefit from different durations/delays (e.g., opacity fades faster than scale). This suggests @keyframes synchronization may be unnecessary for most cases.

Transform Stacking: The base transform value needs to be preserved when animating additional transforms. CSS handles this with space-separated values.
