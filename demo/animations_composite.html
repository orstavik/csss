Composite Keyframe Animations

These demos test "true" composite animations with multiple keyframe stops. Question: Would these benefit from slightly offset timing between properties?

Bounce Effect

Scale and translate at same keyframes. Could scale peak slightly before/after translate?

Bounce
$translateY(0)$scale(1) $go(50%,-30px,1.1) $to(0,1) $animation(0.6s,ease)

Rotation Steps

Rotation with scale at specific points. Would offsetting scale changes improve feel?

Rotate
$rotate(0)$scale(1) $go(33%,120deg,1.2) $go(66%,240deg,0.8) $to(360deg,1)

Slide + Fade

Opacity reaches 100% at 60%, translate at 100%. Better separate or together?

Slide
$translateX(-100px)$opacity(0) $go(60%,opacity(1)) $to(0,1)

Note: This one actually benefits from different timing - opacity completes before movement!

Multi-Property Complex

Transform, color, and opacity with different keyframe stops. Complex composite animation.

Complex
$scale(1)$rotate(0)$bg(#4facfe)$opacity(1) $go(25%,scale(1.3),rotate(45deg)) $go(50%,bg(#fa709a),opacity(0.5)) $go(75%,scale(0.9),rotate(180deg)) $to(scale(1),rotate(360deg),bg(#4facfe),opacity(1))

Fade + Scale In

Classic fade-in with scale. Simple 0-100%, but would it be better if they peaked differently?

Fade In
$scale(0)$opacity(0) $to(scale(1),opacity(1))

Note: This could be improved with a slight overshoot on scale at 80% for a more natural feel.

Complex Path

True composite: rotation and translation must be synchronized to follow a specific path.

Path
$translate(0,0)$rotate(0) $go(25%,translate(50px,-20px),rotate(90deg)) $go(50%,translate(50px,20px),rotate(180deg)) $go(75%,translate(0,20px),rotate(270deg)) $to(translate(0,0),rotate(360deg))

Research Notes: Composite Animations

Question: Would composite animations benefit from offset timing between properties?

Key Observations:

  • Bounce (Demo 1): Scale could reach peak slightly before/after translate for more natural bounce
  • Slide + Fade (Demo 3): Already uses different timing! Opacity completes at 60%, movement at 100%
  • Fade In (Demo 5): Could benefit from scale overshooting at 80% while opacity stays linear
  • Complex Path (Demo 6): This is the rare true composite - rotation and position must stay synced

Conclusion: Most "composite" animations would actually benefit from independent timing per property! True composites (like choreographed paths) are rare. @keyframes may be an overfit to a small edge case.

Better Approach: Allow each property to have its own duration, delay, and easing. For example: $scale(1)$to(1.5)$animation(0.3s,ease-out) $opacity(1)$to(0.5)$animation(0.5s,ease-in)
