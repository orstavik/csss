Relative Calculations & Animation Triggers

Testing relative calculations (*2, +5px) and different animation triggers (:hover, :checked~*)

Relative Calculations

Multiply Border Width

Starting border: 2px. Animation multiplies by 2.

×2
$border(2px,solid,#764ba2) $to(*2)
Relative calc: *2 multiplies current value (2px → 4px)

Scale by Factor

Current scale: 1. Multiply by 1.5.

×1.5
$scale(1) $to(*1.5)
Relative calc: *1.5 multiplies current scale

Add to Translation

Current translateX: 0. Add 50px.

+50px
$translateX(0) $to(+50px)
Relative calc: +50px adds to current value

Hover Trigger

Simplest trigger: :hover pseudo-class.

Hover Me
.box:hover$scale(1)$to(1.2)
:hover is the most straightforward CSS-only animation trigger

Checkbox Hack - Expand

Using :checked~* to reveal hidden content.

This content is hidden by default (max-height: 0, opacity: 0) and revealed when the checkbox is checked. The :checked~* selector targets any sibling after the checkbox.

<input type="checkbox" id="trigger" /> <div class=":checked~*$maxHeight(0)$opacity(0)$to(300px,1)"> Hidden content </div>

Transform Stacking with Checkbox

Base transform preserved, animation adds more transforms.

Stack
/* Base value */ transform: translateY(10px); /* On :checked, stack more transforms */ $translateY(10px) $rotate(0)$scale(1) $to(rotate(180deg),scale(1.3))
Transform stacking: Base translateY(10px) is preserved, animation adds rotate and scale on top

Research Notes

Relative Calculations

Implied currentValue: The arguments before animation shorts become the base values. Relative calculations (*2, +5px, -10%) operate on these base values.

  • *n - Multiply current numeric value
  • +value - Add to current value (with same unit)
  • -value - Subtract from current value
  • /n - Divide current numeric value

Implementation: When processing $to(*2), we detect the relative calc operator (*), extract the current value from the primary short's arguments, and compute the result.

Checkbox Hack (:checked~*)

The checkbox hack provides CSS-only interactive animations:

  1. Hide the checkbox visually: position: absolute; opacity: 0;
  2. Use a label to trigger the checkbox
  3. Target siblings after checkbox: .checkbox:checked ~ * .target
  4. Apply animation/transition styles on the checked state

Pattern in CSSS:

<input type="checkbox" id="toggle" class="trigger" /> <div class=":checked~*$prop(startValue)$to(endValue)"> Animated element </div>

Transform Stacking

CSS transforms can be stacked by separating them with spaces. This is crucial for animations:

  • transform: translate(2px) scale(1);
  • transform: scale(3) translate(2px) scale(1); (all equivalent)
  • When animating, preserve base transform values by prepending them

In CSSS: If a base transform exists, animation keyframes should include it: @keyframes { to { transform: ${baseTransform} ${animatedTransform}; } }
